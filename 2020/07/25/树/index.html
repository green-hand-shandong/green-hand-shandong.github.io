01 树

前中后层次遍历

  • 子树，NLR、LNR、LRN的领会（递归）

  • 有时候把左子树抽象成左孩子，直至叶子变N

  • 前序 NLR，先最左边一道，再是从下到上的子树，再是右边，从上到下的子树

  • 中序 LNR，第一个是最左边的结点，后面拐入右子树（由下到上），

  • 后序 LRN，第一个是

  • 层次遍历 —— 编号序列

中序 + 前/后序 ，得到树

  • 根节点位置特别注意，中序序列，根划分子树。前/后序容易找根。
  • 不断用“根”分割左右子树集合，直至叶子变N

森林转换二叉树

  • 连线，
    • 根节点兄弟之间。
    • 森林子树的同层兄弟之间
  • 删线，
    • 删除根节点和非First Child之间（左孩子右兄弟表示法）
    • 专门看那些不是一个孩子的结点。
    • 有孩子没有兄弟的结点就只剩下左子树
  • 旋转，顺时针 45 °

二叉树转换成森林

  • 删线，删除根节点和右孩子的连线
  • 连线，连接根节点和他的非first Child结点
    • first child的右兄弟们
  • 旋转
  • 注意
    • 森林的第一棵树：根节点 + 左子树
    • 右子树是别的树们。（递归定义）
    • 右边那一道上全是森林的子树根节点

后序遍历森林

  • 相当于中序遍历二叉树
  • 相当于挨个后序遍历森林的子树

前序遍历森林

  • 前序遍历二叉树
  • 前序遍历森林的子树

先根 + 后跟森林，画出森林

  • 先画森林二叉树
    • 先根森林 —— 先根二叉树
    • 后跟森林 —— 中根二叉树
  • 先根找根，中根分割子树，直至叶子变根节点
  • 森林二叉树拆解成森林子树二叉树
    • 删线(所有右枝)—— 加线(根到孩子) —— 旋转

画Huffman 树、

  • 树型不唯一，WPL唯一

  • 频率作权值，

    • 有时候频率是整数、有时小数(累加和==1)

  • 挑最小的两个作孩子合并成新的单位

    • 左小右大 —— 老师判题方便

  • 挑最小的两个的时候，容易脑抽

  • 大的近，小的远

写Huffman编码、

  • 左分支 —— 0
  • 右分支 —— 1
  • 从根到叶子读取、从上到下

求树的WPL

  • Σ 叶子结点WPL
  • WPL = 权值 * 路径长度
  • 路径长度 —— 在第几层

堆

  • 完全二叉树 + 双亲 < 两个孩子 或者 双亲 > 两个孩子
  • 实现优先级队列

插入堆

  • 叶子处，最后一层找
  • 从下向上调整。

删除堆元素

  • 向下调整
  • 删除树根
    • 和尾巴交换，再调整（由根向下）
    • 再删去最后一个位置（此时存的是原根）

建立堆

  • 序列 写成完全二叉树
  • 再调整
    • 从最后一个非叶子结点开始，向前调整

总结

  • 二叉树遍历
  • 遍历序列画树
  • 树和森林转换
  • 森林的遍历
  • 画森林
  • Huffman树和编码和WPL
  • Heap

折半查找的判定树

画判定树

平均查找长度

  • 成功
    • ASL_success == $\frac{1}{结点个数}\sum(层数每层有多少个结点)$
  • 失败
    • ASL_loss ==$\frac{1}{外部结点个数}\sum{[(层数-1)该层的外部结点个数]}$
    • 外部结点 —— 全是叶子结点 n0
      • n0 = n2 + 1

  • mid —— root

关键字
比较次数
  • 注意比较次数，
    • 一次性插入成功 —— 比较一次
    • 和空位比较也算比较次数
  • 线性探测
    • ASL_success = 1/元素个数 * （ 总的比较次数累加_元素个数个 ）
    • ASL_loss = 1/P * 总的比较次数累加_p个
      • 注意空的比较次数是1
      • 不是空的需要比到空（空的也算一次）
      • 注意不是 m个，是 p个（能用函数到达的）
  • 链地址法
    • ASL_s == Σ ( 单链表第K个 * K ) * 1 / 结点个
      • 不算结点表的空结点
    • ASL_l == Σ ( 第K个结点 * (K+1) ) * 1 / p
      • 到NULL比一次的时候，才知道是比较结束
      • 结点表里的空结点算一次

排序方法 & 排序结果

排序结果 到排序方法

  • 有几个找到最终位置的？
  • 冒泡？
    • 是否有交换
    • 最后面是不是有序了
  • 选择？
    • 有序段 + 无序段
    • 无序段 + 有序段
    • 选中之后是交换
  • 快速？
    • pivot一趟定一个
      • 课本上是选择最左边的
    • pivot 前小于pivot，后大于pivot
    • high、low两头跑，
    • 先high，后low，
    • 反常赋给没动的，
    • ==找到pivot

同一序列，不同方法

  • 直接插入排序

    • 最初定第一个是有序序列
    • 然后，选无序序列的第一个插入
      • 从有序序列，后往前找
    • 第一趟之后，有序序列变 2 个

  • 希尔排序

    • 定增量，
    • 间隔为增量的小组，一共有增量个
      • 变小组间有序
    • 增量 == 1 —— 直接插入排序

  • 起泡排序

    • 方向
      • 从右往左，一趟定一个最值到最前面
      • 从左网友，一趟定一个最值到最后面
    • 不断比较两个相邻的元素，调整两个元素的顺序

  • 快速排序

    • pivot选取
      • 一次定一个pivot
    • low、high 两头走。
      • 相等时，是pivot的位置。

  • 直接选择

    • 遍历选取一个最值
    • 和最前面的交换，形成有序序列

  • 归并排序

    • 默认2路归并
      • 每一次相邻的两个有序段（初始为1），归并成一个

  • 堆排序

    • 默认从小到大排序 —— 最大堆/大根堆

      • 建立堆，（完全二叉树）调整（从最大的非叶子结点，编号–）

      • 堆顶（根节点）取，和最后的结点（编号最大的叶子）交换，

        从上到下调整，得到第一趟排序的结果（调整堆完毕的）。

        • 有序的在最后面
        • 调整是和孩子中 大的/小的 交换

稳定性和性能

  • 堆排序
    • 没有额外空间 O(1)
    • 不稳定
  • 快速排序
    • 使用递归工作栈(系统的)，O(log2(n))
    • 不稳定
    • 平均最快
    • 最差 O(n^2)
  • 归并排序
    • 使用辅助数组 —— O(n)
    • 稳定
    • O(n*log2(n))

03 图

DFS、BFS

  • 邻接表

    • 顶点表 —— 所有顶点 + 指向边表的指针
    • 边表 —— 顶点的邻接点们，出边的指向 (出度)
    • DFS —— 顶点 – 边表中一个 – 顶点 – 边表中一个
    • BFS —— 顶点–遍历顶点的边表 – 遍历边表结点的边表

  • 邻接矩阵

  • BFS、DFS生成树

拓扑排序

  • 找入度为0

  • 访问完之后，删去结点和边

  • 再找入度为0的

  • 有环不能拓扑排序

  • 有序的拓扑排序 —— 邻接矩阵是三角

Prim贪心算法

  • 选点法，U 、V-U 、
    • U初始是 一个点
  • 初始最小生成树邻近的最小权值
    • 不能有环
  • 直到 V == U
  • n-1 条边 —— n个结点

Kruskal 贪心算法

  • 选边法
  • 初始点全在U里面
  • 从小到大，选择权值小的
    • 不能有环
  • n-1 条边 —— n个结点

Dijkstra算法

源点 到 其他所有点的最短路径。

  • 路径长度递增次序

  • 手工模拟

    源点 0 到1 到2 到3 …
    距离distance
    上一个点

  • distance 不断更新

    • 小于现在的才更新
    • 同时更新上一个点

  • 每次选择distance最小的作 “起点”

    • 更新他的邻接点的distance

关键路径

  • 关键路径 —— 最长的一条
  • < i , j >
  • Ve( j ) = Ve( i ) + 权值
    • 取大的
    • 从前往后
  • Vl( i ) = Vl( j ) - 权值
    • 取小的
    • 从后往前

总结

  • DFS、BFS
  • 拓扑
  • Prim
  • Kruskal
  • Dijkstra
  • 关键路径

04 线性结构

栈、FILO

入栈顺序到出栈顺序

  • 暗含着可以边出边进
  • 卡特兰数 —— 所有可能的合法的，数目
  • 以 X 开头的
  • 如果编号大的先出来了 —— 其后的是逆序 !!!!

符合某规则的出栈序列

  • 变量名：字母下划线开头
    • 数字不行

队列假溢出

  • 队头出，队尾进

    • 队尾元素指向元素
    • 队头指向元素前空的

  • 拓展：双端队列

  • 循环队列解决：%

    • 牺牲一个空间来区分队列 空 / 满
      • 标志位tag法（空/非空）、计数器

  • 牺牲

    • 满 —— (rear +1) % max == front
    • 空 —— rear == front

  • 标志位

    • 满 —— tag == 1 && front == rear
    • 空 —— tag == 0

栈：中缀转后缀

后缀表达式，无括号、不用考虑优先级别

  • optr一个栈，存运算符
    • 栈外优先级高 —— 入栈
    • 栈外有限级低 —— 出栈
  • 数字直接写入后缀表达式
  • 最后把栈中的运算符弹出
  • 技巧
    • 加括号 —— 移动运算符 —— 删除（）

栈：求值后缀表达式

  • 扫描，不用考虑括号和运算符优先级别
  • 数字，压栈
  • 运算符，两个数字出栈运算得到结果，
    • 结果再入栈

字符串 next、nextval

  • next
    • 子串 —— 首、尾串
      • 首串尾串不能完全重叠
    • 1,2对应是 -1 和 0
  • nextval
    • 第一个是 -1
    • 和下标是next的字符比较
      • 相同 —— 继承 nextval
      • 不同 —— 使用自己的next
  • BF算法
    • i * j 次匹配
    • i，j 都回溯
  • KMP
    • i 不回溯， 只有 j 回溯
      • j 回溯到 nextval 下标位置

总结

  • 栈，进、出序列
  • 队列判断 空满
  • 栈： 后缀
  • next 和 nextval

06 AVL平衡二叉树（最佳二叉排序树）

  • 树退化成单支树，优势不复存在

  • 越矮越丰满查找效率越高

  • 满/完全二叉树最好，但是难做 —— 转向AVL

  • AVL：所有结点，bf <=1

    • 不一定是完全二叉树

    • 平衡因子 bf ：结点左右子树的高度差

    • 最大高度证明是：

    • $$
      1.44*log(N+2)-1.328
      $$

调整平衡

  • 由于会调整，导致插入的元素不一定作叶子

LL型调整

  • 找到值中间那个做根节点

RR型调整

  • 找到值中间那个作根节点

LR型调整

  • 找到值中间那个作根节点
  • 最后要符合L<N<R

RL型调整

  • 同LR

构造

  • 构造排序二叉树那一套
  • 随时注意不平衡
    • 调整 —— 四个类型
    • 找bf >1 的结点（最矮的失衡结点开始）

判断平衡二叉树

  • 所有结点 bf < = 1

根节点失衡

  • 沿着最长的分支找3个结点
  • 值处于中间的 —— 作根
  • L<N<R

AVL树是一棵排序树

  • 中序遍历

    • L<N<R —— 升序
    • L>N>R —— 降序
